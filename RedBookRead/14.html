Node

在介绍DOM中节点之前，应该首先介绍Node。上面也说了DOM节点每个都有nodeType和nodeValue属性，这俩属性保存着有关节点的信息。

Node中还有一个childNodes属性，值为NodeList（子节点的类数组）。

Node中有nextSibling和previousSibling属性，指向位于同一DOM层中后一个节点和前一个节点。如果没有的话，值为null。

Node中有appendChild方法，用于在childNodes列表末尾插入节点。

属性/方法 返回值 作用
Node.prototype.childNodes（可读） NodeLists 获取this下的子节点
Node.prototype.lastChild（可读） node 获取this下最后一个子节点
Node.prototype.firstChild（可读） node 获取this下第一个子节点
Node.prototype.nextSibling（可读） node 获取当前dom树下 this后同胞节点
Node.prototype.previousSibling（可读） node 获取当前dom树下this前同胞节点
Node.prototype.appendChild( node ) newNode 将node插入到this的childNodes的末尾
Node.prototype.insertBefore(newNode, oldNode) newNode 将newNode插入到this的childNodes中的oldNode前面
Node.prototype.replaceChild(newNode, oldNode) oldNode 将oldNode替换成newNode
Node.prototype.removeChild(node) node 移除node节点
Node.prototype.cloneNode(boolean) node 如果为true，则深拷贝node，否则浅拷贝node
Node.prototype.normalize() undefined 处理文档子树中的文本节点

Document

Document指向整个DOM节点树，代表整个HTML页面，document 是 window 的一个属性。所以其实我们每次些document…. 其实省略了window.。

Document的nodeType为9，nodeName为“#document”，nodeValue为null,parentNode为null，ownerDocument为null。子节点可以为DocumentType（最多一个）、Element（最多一个）、ProcessingInstruction、Comment。

Document.documentElement就是html元素，对应子节点就是Element。

Document.doctype就是<! Doctype>文档声明，对应字节带你就是DocumentType。

comment就是文档的注释。

document文档写入功能：

方法名 说明 用法
write(str) 将字符串写入网页 常用于动态包含外部资源
writeIn(str) 在字符串末尾追加一个换行符\n 常用于动态包含外部资源
open() 打开网页 打开网页
close() 关闭网页 关闭网页

Element

所有的html元素都是通过HTMLElement类型表示。每个属性对应下列属性值之一：

  1. id 唯一标识符
  2. title 包含元素的额外信息
  3. lang 元素内容的语言
  4. dir 语言的书写方向
  5. className 相当于class属性，后面也有对应的属性classList。不使用class是因为class是ES的关键字。

首先，调用MutationObserver构造函数并传入一个回调函数作为参数创建一个MutationObserver实例。

第二步，调用observe方法，将该实例与DOM节点绑定，observe接受两个参数，第一个参数是DOM节点，第二个参数叫做MutationObserverInit对象。这个对象用于控制哪些方面的变化。

下面表格说明MutationObserverInit对象中的合法属性。

属性 说明
subtree 布尔值，表示除了目标节点，是否观察目标节点的后代。默认为false，则之观察目标节点的变化，为true则观察整个子树节点的变化
attributes 布尔值，默认为false，表示是否观察目标节点的属性变化
attributeFilter 字符串数组，表示要观察哪些属性变化，将这个值设为true，会将attributes设为true
attributeOldValue 布尔值，表示MutationRecord是否记录变化之前的属性值
characterData 布尔值，表示修改字符属性是否触发回调函数。默认为false
characterDataOldValue 与attributeOldValue类似
childList 表示修改目标节点的子节点是否触发回调函数。

表格中提到了MutationRecord，这个东西是回调函数的参数，包含的信息包括发生了说明变化，以及哪一部分受到了影响

因为回调执行之前可能同时发生多个满足观察条件的事件，所以每次执行回调都会传入一个包含按顺序入队的MutationRecord实例的数组。

MutationRecord也有许多属性。下面用一个表格来展示：

属性 说明
target DOM节点
type 表示变化的类型
oldValue 表示旧值，当characterDataOldValue或者attributeOldValue设为true时有值，否则为null
attributeName 对于attributes: true，保存的是被修改的属性名
attributeNamespace 对于使用了命名空间的”attributes“类型的变化，保存的是被修改的属性名
addedNodes 对于”childList”类型的变化，返回添加节点的NodeList
removedNodes 对于”childList”类型的变化，返回删除节点的NodeList
previousSibling 对于”childList”类型的变化，返回变化节点的前一个同胞Node
nextSibling 对于”childList”类型的变化，返回变化节点的后一个同胞Node
