JavaScript(10)

Selectors API

在DOM1中，获取DOM节点的方法准确来说只有两个：getElementById(id) 和 getElementsByTagName(tagName)，而在jQuery中，完全以CSS选择符查询DOM获取元素的引用。

而Selectors API Level1中提供了两种方法，他的表现就如同jQuery中的$()查询符。querySelector(CSSDescription)，querySlectorAll(CSSDescription)。第一个方法返回第一个匹配的DOM元素，第二个方法返回匹配的NodeList。

在兼容浏览器中，暴露这俩方法的类型是Document，Element。

还有一个方法matches()我倒是从来没用使用过，接受的参数也是CSSDescription。不过返回的是boolean，如果元素匹配CSSDescription，为true，否则为false。

HTML5

CSS 扩展

1、 getElementByClassName

2、 classList

H5之前操作类名是className字面量形式的，H5提供了一个属性ClassList实现class类的添加、删除和替换。

提供了如下方法。

方法 说明
DOM.classList.add(value) 添加
DOM.classList.remove(value) 删除
DOM.classList.toggle(value) 如果有则删除，如果没有则添加
DOM.classList.contains(value) 返回boolean，表示给定的value是否存在

3、document.activeElement 始终包含当前拥有焦点的DOM元素。

4、dataset

在标签上使用data-自定义数据属性，可以通过DOM.dataset.propName来访问。自定义数据的数据类型都是字符串类型。

5、Element类型

新增几个属性，firstElementChild，lastElementChild，nextElementSibling，previousElementChild，

children。对比fristChild，lastChild，nextSibling，previousSibling，childNodes，很明显返回的都是Element类型的节点。

DOM2&&DOM3

DOM2和DOM3都是在DOM1定义的底层结构上加入更多的交互能力。

DOM2在Node的原型对象上新增了命名空间这一概念。当然也衍生了许多对应的方法。不过大体上都是原先的方法后缀-NS，接受的第一个参数改成了namespace，有点C++的意思。

DOM2新增了isSameNode()和isEqualNode()方法。故名思意，如果两个节点只是属性一样，并不是值一样调用isSameNode为false，调用isEqualNode为true。
