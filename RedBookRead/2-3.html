第三章

1、重复声明

在js中，使用var声明重复声明同名变量会忽略后面的声明。使用let重复声明同名变量会报错。

注意：这块容易忽略的点是，使用var声明后，又使用let声明同样会爆出SynatxError的错误。这应该很好理解，但是使用let声明后，再使用var声明，也会报错！

尝试给一个解释：使用var和let声明变量，只是给变量一个状态。var没有块作用域，let有块作用域，一个变量不可能同时存在两个作用域中。当给定一个变量一个状态后，就不能改变状态了。

网上看到另一种解释：let具有TDZ，var可以声明提升，在let声明前就使用了这个变量名，也会报错。

这块可能得结合ES6的一些知识，目前还没了解到，等了解到了再来补充。

2、Number八进制

我看红宝书上说到，对于八进制字面量，以0开头。十六进制字面量，以0x开头。

ES6中，使用0o来表示八进制，严格模式下前缀0会报错。

使用0o前缀就对了。

3、parseInt和parseFloat

这两个函数在你不知道的js里面叫做解析，Number被成为转换。主要区别有：

  1. Number(‘12ab’) => NaN , parseInt(‘12ab’) => 12;
  2. Number(‘’) => 0, parseInt(‘’) => NaN;
  3. Number(‘12.5’) => 12.5 , parseInt(‘12.5’) => 12
  4. Number转换的一个特殊情况，Number(‘\n’) = 0

这是一些转换的差异，对parseInt函数，只接受字符串，如果不是字符串调用toString()，否则返回NaN。parseInt接受两个参数，第二个参数表示用多少进制解释，如果不写的话默认以字符串前缀来解释。

根据上图，很明显当我parseInt(‘023’) 返回的居然是23，这还是很难理解的…MDN上解释字符串以0开头被假定为八进制或者十六进制，具体是哪个进制取决于实现，很显然，Chrome是十进制。因此，在使用parseInt时，一定要指定进制！

相比较于parseInt,parseFloat就简单的多，1、只能接受一个参数，2、如果没有小数点后的数就直接返回整数，3、用十进制来解释，4、0开头的字符串会忽略0。

4、模板字面量与标签函数

模板字面量和字符串字面形式不同的点有：

  1. 声明方式不同，模板字面量``声明，字面形式使用单引号，双引号。
  2. 模板字面量有一种特殊的调用形式。fn`string`
  3. 模板字面量支持${}插值引入变量的值。${}叫做标签占位符。插入的值调用toString方法强行转换成字符串。
  4. 模板字面量可以跨行定义字符串。

5、Symbol类型

Symbol 是ES6新增的数据类型。symbol是原始值，Symbol实例是唯一的，不可变的。Symbol 不可以当作构造函数，而且没有字面量语法。

上面图片可以验证symbol的唯一性。

全局符号注册表

不同的部分需要共享和重用符号实力，那么可以用字符串作为键，在全局符号注册表中创建并重用符号。

使用Symbol.for()方法进行操作。

第一次调用Symbol.for(‘foo’) 先查询全局符号注册表中的键是否有‘foo’，如果有的话，拿过来就用，如果没有的话，构建一个以‘foo’为键的符号实例。

第二次调用也会先查询是否有‘foo’，有，拿过来使用即可。

这个时候两个变量指向同一个内存，所以二者相等。

很明显，Symbol(‘foo’) 和 Symbol.for(‘foo’)即使使用相同的符号描述，他俩并不相同。

这就有点对象相等的概念了。
