强制类型转换

将值从一种类型转换成另一种类型通常称为类型转换，隐式的情况称为强制类型转换，
类型转换发生在静态类型语言的编译阶段，而强制类型转换则发生在动态类型语言的运行时。

关于Array

构造函数Array(..)不必须要求带new关键字，不带时，他会被JS引擎自动补上。
数组中并没有预设长度的概念，let arr = Array(3) 这样创建出来的只是一个空数组，只是length属性被设置成指定的值。当数组实际长度大于length时，length也会随着数组实际长度的改变而改变。

注：

  1. 当空数组执行map时会失败，因为数组中并不存在任何单元，所以map无从遍历。
  2. 当空数组执行join时会成功，因为join首先假定数组不为空，然后通过length属性值来便利其中的元素。
  3. 永远不要创建和使用空单元数组！

强制类型转换

将值从一种类型转换成另一种类型通常称为类型转换，隐式的情况称为强制类型转换，
类型转换发生在静态类型语言的编译阶段，而强制类型转换则发生在动态类型语言的运行时。

显式类型转换

ToString规则

toString()

toString() 会将null转换为“null”,undefined转换成“undefined”,数字的字符换则遵顼通用规则，对于普通对象(即typeof 返回值是object)，Object.prototype.toString()返回内部属性[[Class]]的值。

JSON.stringify()

JSON.stringify(..) 可以将JSON对象序列化。JSON字符串转化话和toString(..)的效果基本相同，但序列化的结果总是字符串。所有安全的JSON值能使用JSON.stringify(..)字符串转换。
undefined、function、symbol （ES6+）和包含循环引用（对象之间相互引用，形成一个无限循环）的对象都不符合 JSON 结构标准，支持 JSON 的语言无法处理它们。
如果要对含有非法JSON值的对象做字符串化，需要自定义toJSON(..)方法返回一个安全的JSON值。JSON字符串化会首先调用该方法，然后用它的返回值来序列化。

ToNumber 规则

  1. true => 1 , false => 0 , undefined => NaN , null => 0 ，[] => 0 , {} =>NaN
  2. 检查该值是否有valueOf方法，有就返回基本类型值，使用该值进行强制类型转换，没有就使用toString()返回值字符串来强类型转换。

ToBoolean 规则

  1. 假值：undefined，null，false，0，NaN,””。
  2. 假值对象:对象都是真值，假值对象是在语法基础上自己创建了一些外来值，这些值就是“假值对象”。常见的是document.all。可以通过document.all来判断是否是版本IE。
  3. 真值：假值之外的值都是真值。空对象，空数组，空函数都是真值。

ToPrimitive 规则

Symbol.ToPrimitive 是一个内置的Symbol值，它是作为对象的函数值属性存在的，当一个对象转换为对应的原始值时，会调用此函数。该函数调用时，会被传递一个字符串参数hint，表示要转换到的原始值的预期类型，hint = [“number”|”string”|”default”]
为了将对象强制类型转换为基本数据类型值，首先调用valueOf(..)方法，如果有并且返回基本类型值，就会用该值进行强制类型转换，如果没有，就调用toString(..)方法如果有并且返回基本类型值，就会用该值进行强制类型转换。如果均不返回基本类型值，就会产生TypeError错误。

显式强制类型转换

字符串和数字之间显式类型转换
String(..)和Number(..)

toString(..)显示转换。不过对基本类型值不适用，所以JS引擎会为基本类型值封装对象，在对象上调用toString(..)。

  1. +c 是+运算符的一元形式。显示将字符串转化成数字。
  2. ~ 运算符：字位运算符只适应32位整数，~x = -(x+1) x=-1时，~x = 0
  3. ~~运算符，字位截除，返回整数部分。因为数位运算符只能在32位有效，~~可以截取32位数字。
    显示解析数字字符串
    显示解析数字字符串：parseInt、parseFloat，这两个方法接受两个参数，第一个是要转换的字符串，第二个参数是指定转换的进制。
    解析（即parseInt、parseFloat）允许字符串中含有非数字字符。Number不允许。
    ES5之前的parseInt(..)如果没有第二个参数来指定转换的禁止，会根据字符串第一个字符来自行决定转换的进制。如果没有第二个参数来指定转换的参数，会根据字符串第一个字符来自行决定转换成的进制。
    js中科学计数法
    当数值小于1而且小数点后面带有6个0以上的浮点数值和整数位数字多于21位时，js引擎会自动将数值转换成科学计数法的表现形式。
    数字运算中的精度缺失问题
    这个问题有一个受众很广的例子，0.1+0.2！=0.3
    下面分析一下原因
    计算机中数字是以多位二进制进行存储的，js中采用的是IEEE 754的双精度标准。也就是说0.1 这样的数值用二进制表示就会发现无法整除，0.1 用二进制转换后的结果为0.0 0011 0011 …..由于计算机的存储数位有限，最后计算机后舍弃后面的数值，所以我们只能得到一个近似值。
  4. 1 和 0.2 这个式子中，0.1和0.2都是近似表示的，在他们相加的时候，两个近似值都进行了计算，导致最后得到的结果是0.30000000000000004,对于js来说不够近似于0.3，于是就出现了0.1+0.2！=0.3
显示转换为布尔值

Boolean 是显式的ToBoolean强制类型转换。

  1. ！一元运算符显式将值转换成布尔值，同时还将真值反转。所以最常用的转换成布尔值是!!。

隐式强制类型转换

字符串和数字之间的隐式强制类型转换
  1. 如果某个操作数是字符串或能通过一些步骤转换成字符串，+将进行拼接操作。如果一个操作数是对象，首先调用ToPrimitive抽象操作，该操作再调用[[DefaultValue]],以数字作为上下文。
  2. 将数字和空字符串相加将数字转化成字符串。2+’’，根据ToPrimitive抽象从左首先调用valueOf方法，然后通过ToString抽象操作将返回值转换成字符串。而String(2)则会直接调用ToString()。
  3. 字符串强制转换成数字： ‘3,14’ - 0 或者’3.14’ * 1 等等数字运算符。
布尔值到数字的隐式强制类型转换
  1. 隐式强制类型转换为布尔值
    if( .. )、 for( .. ; .. ; .. ) 、while( .. ) 、? : 、||等非布尔值会被隐式强制类型转换为布尔值，遵循前面介绍过的ToBlloean抽象操作规则。
  2. && 和 || 操作数选择器
  • && 第一个数为真返回第二个数的值，第一个数为假返回第一个数的值。
  •  ||   第一个数为真返回第一个数的值，第一个数为假返回第二个操作数的值。
  1. 符号的强制类型转换
    符号不能够被强制类型转换为数字，但可以被强制类型转化为布尔值（显示隐式都为true）。
  2. == 和 ===（两边都会转换成数字类型进行比较）
    == 允许在相等比较中进行类型转换，=== 不允许。
    对象在比较中调用ToPrimitive抽象规则，先调用valueOf,对返回的结果调用toString。
==数值相等的算法细节

1、如果x不是正常值（比如抛出一个错误），中断执行。
2、如果y不是正常值，中断执行。
3、如果Type(x)与Type(y)相同，执行严格相等运算x === y。
4、如果x是null，y是undefined，返回true。
5、如果x是undefined，y是null，返回true。

6、如果Type(x)是数值，Type(y)是字符串，返回x == ToNumber(y)的结果。

7、如果Type(x)是字符串，Type(y)是数值，返回ToNumber(x) == y的结果。
8、如果Type(x)是布尔值，返回ToNumber(x) == y的结果。
9、如果Type(y)是布尔值，返回x == ToNumber(y)的结果。
10、如果Type(x)是字符串或数值或Symbol值，Type(y)是对象，返回x == ToPrimitive(y)的结果。
11、如果Type(x)是对象，Type(y)是字符串或数值或Symbol值，返回ToPrimitive(x) == y的结果。

12、返回false。
总结：
1、如果有null或者undefined，返回false
2、如果有Boolean将Boolean转换成Number
3、 如果其中一个是对象，执行ToPrimitive抽象规则。（先调用valueOf，如果没有valueOf或者返回的不是基本数据类型,调用tostring）
4、如果一个是数字，一个是字符串，将字符串转化成数字。
5、如果x,y不正常，中止执行，如果x,y为null和undefined，返回true
6、根据规范，比较a <= b时会比较 b < a,并将结果反转。
