BOM

BOM是浏览器对象模型（Browser Object Model）的缩写，他允许JavaScript与浏览器直接进行对话。BOM不存在官网标准，现代的浏览器几乎已经实现了JavaScript交互相同的方法和属性，因此她经常作为BOM的方法和属性被提到。

window对象简介

所有的浏览器都支持window对象，它代表浏览器的窗口，所有全局JavaScript对象，函数和变量自动成为window对象的成员。全局变量就是window对象的属性，全局函数就是window对象的方法，DOM中的document对象也是window对象的属性。

window窗口方法

window中innerWidth、innerHeight以像素为单位返回尺寸，innerHeight返回浏览器窗口的内高度，innerWidth返回浏览器窗口的内宽度。还有一些如window.open()、window.close()、window.moveTo()、window.resizeTo( )方法，可以用来调整窗口。

window.open()

该方法打开的新窗口对象的引用。如果调用失败，返回值会是 null 。如果父子窗口满足“同源策略”，你可以通过这个引用访问新窗口的属性或方法。
strUrl
新窗口需要载入的url地址。strUrl可以是web上的html页面也可以是图片文件或者其他任何浏览器支持的文件格式。
strWindowName
新窗口的名称。该字符串可以用来作为超链接 <a> 或表单 <form> 元素的目标属性值。字符串中不能含有空白字符。注意：strWindowName 并不是新窗口的标题。
strWindowFeatures
可选参数。是一个字符串值，这个值列出了将要打开的窗口的一些特性(窗口功能和工具栏) 。 字符串中不能包含任何空白字符，特性之间用逗号分隔开。参考下文的位置和尺寸特征。
注意：调用window.open()方法以后，远程 URL 不会被立即载入，载入过程是异步的。（实际加载这个URL的时间推迟到当前脚本块执行结束之后。窗口的创建和相关资源的加载异步地进行。）

window.close()

该方法只能有window.open()打开的新窗口来调用，作用是关闭一个有window.open()方法打开的窗口。
如果窗口不是由window.open()打开，调用该方法控制台会报错

window.location 对象

window.location对象可以不带window前缀书写。
共有如下几种常用的方法：href(返回当前页面的URL)、hostname(返回web主机的域名)、pathname(返回当前页面的路径或者文件名)、protocol(返回使用的web协议，如：http、https)、assign(加载新文档)。
location.href = location.hostname + location.pathname

window.history 对象

window.history 对象可以不带window书写，为了保护用户的隐私，JavaScript访问此对象存在限制。
一些常用的方法如：history.back()，history.forward()，history.go()。
history.back() 等同于在浏览器中点击前进按钮，返回上一个页面。
history.forward() 等同与在浏览器点击后退按钮，进入下一个页面。
history.go()载入到绘画历史中的某一特定页面，通过与当前页面相对位置来标志（当前页面为0）。

HTML5赋予的新特性

从HTML5开始，提供了对history栈内容的操作。HTML5引入了history.pushState()和history.replaceState()方法，他们可以添加和修改历史记录条目。
使用 history.pushState() 可以改变referrer，它在用户发送 XMLHttpRequest 请求时在HTTP头部使用，改变state后创建的 XMLHttpRequest 对象的referrer都会被改变。

JavaScript 弹出框对象

JavaScript 有三种类型的弹出框：警告框、确认框和提示框。
警告框如果要确保信息传递给用户，通常会使用警告框。当警告框弹出时，用户将需要单击“确定”来继续。window.alert()可以不带window前缀来写。

确认框如果您希望用户验证或接受某个东西，则通常使用“确认”框。当确认框弹出时，用户将不得不单击“确定”或“取消”来继续进行。如果用户单击确定，该框返回 true。如果用户单击取消，该框返回false。window.confirm()可以不带window前缀来写。

提示框如果您希望用户在进入页面前输入值，通常会使用提示框。当提示框弹出时，用户将不得不输入值后单击“确定”或点击“取消”来继续进行。如果用户单击“确定”，该框返回输入值。如果用户单击“取消”，该框返回 NULL。window.prompt() 方法可以不带 window 前缀来编写。
